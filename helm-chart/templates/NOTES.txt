Kibana SSO Proxy has been deployed!

{{- if .Values.ingress.enabled }}

Access the proxy at:
{{- range $host := .Values.ingress.hosts }}
  {{- range .paths }}
  http{{ if $.Values.ingress.tls }}s{{ end }}://{{ $host.host }}{{ .path }}
  {{- end }}
{{- end }}

{{- else if contains "NodePort" .Values.service.type }}

Get the application URL by running:
  export NODE_PORT=$(kubectl get --namespace {{ .Release.Namespace }} -o jsonpath="{.spec.ports[0].nodePort}" services {{ include "kibana-sso-proxy.fullname" . }})
  export NODE_IP=$(kubectl get nodes --namespace {{ .Release.Namespace }} -o jsonpath="{.items[0].status.addresses[0].address}")
  echo http://$NODE_IP:$NODE_PORT

{{- else if contains "LoadBalancer" .Values.service.type }}

NOTE: It may take a few minutes for the LoadBalancer IP to be available.
      Watch status with: kubectl get --namespace {{ .Release.Namespace }} svc -w {{ include "kibana-sso-proxy.fullname" . }}

  export SERVICE_IP=$(kubectl get svc --namespace {{ .Release.Namespace }} {{ include "kibana-sso-proxy.fullname" . }} --template "{{"{{ range (index .status.loadBalancer.ingress 0) }}{{.}}{{ end }}"}}")
  echo http://$SERVICE_IP:{{ .Values.service.port }}

{{- else if contains "ClusterIP" .Values.service.type }}

The proxy is available internally at:
  {{ include "kibana-sso-proxy.fullname" . }}.{{ .Release.Namespace }}.svc.cluster.local:{{ .Values.service.port }}

For local testing, run:
  kubectl --namespace {{ .Release.Namespace }} port-forward svc/{{ include "kibana-sso-proxy.fullname" . }} 3000:{{ .Values.service.port }}

Then access: http://localhost:3000/login

{{- end }}

Configuration:
  Provider: {{ .Values.config.OIDC_PROVIDER }}
  Kibana URL: {{ .Values.config.KIBANA_PUBLIC_URL }}

Endpoints:
  /login       - Initiate SSO login
  /auth/logout - Logout and clear session
  /auth/user   - Get current user info
  /health      - Health check

For more information, visit:
  https://github.com/nativesre/kibana-sso-proxy
